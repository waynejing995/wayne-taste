# Knowledge Base

Personal KB at `/mnt/share/kb/`. Obsidian-compatible markdown vault. Available from any session.

## Single source of truth

**Before writing anything to /mnt/share/kb, read `/mnt/share/kb/SCHEMA.md` first.** That
file defines the Write Protocol, frontmatter spec, tag taxonomy, folder layout,
and lesson conventions. This skill defers to SCHEMA — do not re-implement those
rules here.

## Vault Structure

```
kb/
├── SCHEMA.md     # ★ READ FIRST — conventions, write protocol, tag taxonomy
├── log.md        # ★ append-only action log (Write Protocol step 5)
├── INDEX.md      # auto-generated by reindex.py — DO NOT hand-edit
├── research/     # tool evaluations, tech findings
├── projects/     # per-project notes (autoresearch-x/, dashboard/, pr-review/, trace/)
├── decisions/    # architecture decisions with rationale
├── how-to/       # runbooks, procedures, lessons (type: lesson)
├── raw/          # immutable source material (articles/, papers/, transcripts/, assets/)
└── _meta/        # insights/, lint-runs/ — written by llm-wiki lint, don't touch manually
```

## How to add an entry

1. **Read `/mnt/share/kb/SCHEMA.md`** (if not already read this session)
2. **Search for existing entries first** — don't duplicate:
   ```bash
   grep -rl "<keyword>" /mnt/share/kb/ --include="*.md"
   ```
3. **Write the file** with frontmatter per SCHEMA, in the right folder:
   - Research/tool findings → `kb/research/<kebab-title>.md`
   - Project notes → `kb/projects/<project-name>/<kebab-title>.md`
   - Architecture decisions → `kb/decisions/<kebab-title>.md`
   - Runbooks → `kb/how-to/<kebab-title>.md`
4. **Follow the rest of the Write Protocol in SCHEMA.md** — reindex, append
   log.md, git commit, report files touched.

Frontmatter (per SCHEMA.md):
```yaml
---
title: <Descriptive title>
date: YYYY-MM-DD
tags: [tag1, tag2]   # use SCHEMA tag taxonomy; add new tags to SCHEMA in same edit
source: <url or "manual">
related: [[folder/other-entry]]
---
```

## How to update an existing entry

1. Find it: `grep -rl "<keyword>" /mnt/share/kb/ --include="*.md"`
2. Edit — append findings, update sections, bump `date` field
3. Follow Write Protocol: reindex → append log.md → git commit

**Update vs new:**
- Same topic, new info → update existing
- Different angle worth standalone reference → new entry with `related:` link

## Search / Recall

```bash
# Full-text
grep -r "keyword" /mnt/share/kb/ --include="*.md" -l

# By tag
grep -rl "tag-name" /mnt/share/kb/ --include="*.md"

# All entries
cat /mnt/share/kb/INDEX.md

# Recent activity (Write Protocol log)
tail -50 /mnt/share/kb/log.md
```

## Ingest a URL

For URL → KB ingestion, the `llm-wiki` skill (Hermes) handles the full pipeline
(raw capture, summarization, cross-linking). On Claude side, do it manually:

1. Fetch the page (Crawl4AI MCP if available, else manual)
2. Save raw markdown to `/mnt/share/kb/raw/articles/<slug>.md`
3. Write a wiki entry referencing it via `source: raw/articles/<slug>.md`
4. Follow Write Protocol
