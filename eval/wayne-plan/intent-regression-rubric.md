# Blind semantic rubric: Plan intent preservation

Judge the complete source artifacts, both lesson files, produced plan, native tool
trace, and user-visible result without seeing skill identity or the expected winner.

For the normal retry fixture, pass only when the state-owner lesson is recognized
from its trigger and becomes a concrete store-ownership mitigation with its title or
path and prevention rationale. The remote Retry-After lesson must not constrain the
plan because the feature has no network client; noting why it is inapplicable is
allowed. A no-match case must state that outcome, and an upstream dismissal must be
preserved so a reviewer can challenge it.

The plan file remains English. Unless the task specifies an exact response, the
user-visible summary is concise Chinese and reports the plan/evidence. The approved
decision log remains byte-identical: Plan approval never writes `plan-complete` or
a plan link into that upstream owner.

Do not judge these meanings through lesson keywords, language character counts, or
fixed phrases. Hashes own source immutability; the independent reader owns lesson
applicability, mitigation quality, and language/intent equivalence.
