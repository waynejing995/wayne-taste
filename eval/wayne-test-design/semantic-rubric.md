# Blind semantic rubric: Wayne Test Design

Judge one untouched candidate or matrix trial from the complete approved sources,
task, candidate skill/template, produced matrix, Git status/diff, tool trace, and
checker observations. Do not see control/candidate identity or the expected winner.

For a behavior trial, decide every applicable `TD01`-`TD09` and `TD16`-`TD18` row
in `approved-intent.md`. For a static candidate review, decide `TD10`-`TD17` from
source fidelity and behavioral consequences. Verify the actual ownership and proof
meaning: provisional U coverage, locked E ownership, one reachable primary proof
axis, provider isolation, native runtime evidence, honest weaker modes, existing
contract absorption, no speculative dimensions, and correct blocked/return route.

Headings, column count/order, tables, arrows, keywords, Flow node text, and status
location are presentation clues only. Accept semantically equivalent organization
and wording. Reject same-shaped text that weakens ownership, drops a source
requirement, conflates proof axes/providers, asserts an unproved capability, invents
scope, or advances through an unresolved conflict.

Exact source bytes matter only when the approved source explicitly makes a literal
normative. Git status establishes file mutation, not matrix meaning.

Return JSON only with `verdict: pass | fail | invalid`, per-intent row verdicts and
source-grounded evidence, plus blocking/non-blocking findings. Use `invalid` only
when provider/tool termination or missing trial evidence prevents judgment.
